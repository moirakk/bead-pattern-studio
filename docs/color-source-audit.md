# 色号来源交叉验证

## 当前结论

当前内置色卡 `A01` 到 `A36` 是产品原型用演示数据，不是官方 Artkal、Hama、Perler/Kawada 或任何店铺的真实色卡。

因此：

- 演示色卡只能用于测试算法和 UI。
- 不能直接用于备豆、下单、材料包、公开售卖图纸。
- 真实项目必须导入店铺或品牌提供的 `code,name,hex` 色卡。

## 已核对的实际色号体系

- Artkal 官方 5mm Midi 使用 `S01`、`S02` 等 S 系列色号；2.6mm Mini 使用 `C01`、`C02` 等 C 系列色号。
- Hama 官方色卡使用数字色号，并说明 Mini、Midi、Maxi 的颜色代码共用。
- 日本 Perler/Kawada 官方提供独立颜色列表和 PDF 色卡。

这些体系都不等于当前原型里的 `A01 Snow White`、`A02 Ivory` 等演示数据。

## 上线质量门槛

1. 每个内置正式色卡必须记录来源、品牌、系列、版本日期。
2. 官方或店铺色卡必须和原始来源抽样核对至少 10 个颜色编号。
3. 如果色卡来自用户上传，导出文件必须显示“用户导入色卡，请以店铺原始数据为准”。
4. 如果使用演示色卡，导出文件必须显示“非官方占位色号，不可直接用于采购/制作”。
5. 未来后台应保存色卡版本 ID，保证旧图纸能追溯当时使用的色号表。
