# 后端与核心架构审计

审计日期：2026-07-19

## 当前真实边界

当前版本是本地优先的 Web / Capacitor 应用，不包含账号或云端业务数据库。

| 能力 | 当前实现 | 数据位置 |
| --- | --- | --- |
| 图片转换与编辑 | 浏览器 Canvas + TypeScript 纯函数 | 设备内存 |
| 作品保存 | IndexedDB，失败时回退 localStorage | 当前设备 |
| 备份与恢复 | `.beadproject` 版本化 JSON | 用户导出的文件 |
| 社区发现 | 内置示例数据 | 应用包 |
| 点赞、收藏、发布草稿 | 本地状态 / localStorage | 当前设备 |
| PNG / PDF | Canvas 与本地 PDF 生成器 | 当前设备 |
| 服务端 API | 仅 `/api/health` 运行状态检查 | 无业务数据 |
| Cloudflare D1 / R2 | 未启用 | `.openai/hosting.json` 均为 `null` |

因此，删除 App、清理浏览器数据或更换设备时，作品不会自动同步。当前隐私说明中的“图片不上传、无需账号”与实现一致。

## 本次已核对并加固

- 图纸核心限制为最大 `500 x 500`，拒绝零尺寸、非整数尺寸、无效色数和重复色号。
- RGB、HEX、图像调整参数均检查有限值，避免损坏数据静默变成黑色或 `NaN`。
- 店铺 CSV 支持 BOM、带逗号或双引号的名称，限制 2 MB / 1000 色，并拒绝重复色号。
- 备份限制 96 MB / 100 个作品，校验尺寸、色卡、Lab 派生值、色号与格子的一致性及重复作品 ID。
- `.beadproject` 默认导出紧凑 v2，使用色号字典和二进制格子数据；旧 v1 文件仍可导入。
- IndexedDB 在成功和异常路径都会关闭连接；事务失败后才回退 localStorage。
- 新作品使用唯一 ID；重新打开作品再保存会更新原作品，并保留分类与复刻来源。
- 社区图纸模型保留每格色号；复刻不再重新压缩为 48 色，也不会因相同 HEX 丢失真实色号。
- PDF 限制 100 页并验证 JPEG 数据及页面尺寸。
- 图片限制 30 MB / 8000 万像素，上传对象 URL 在成功和失败路径均释放。
- Next.js 升级到 16.2.10，并将其 PostCSS 覆盖为 8.5.19；生产依赖 `npm audit` 为 0 个已知漏洞。

## 仍然存在的产品级限制

1. IndexedDB 是设备级存储，不具备账号同步、跨设备恢复和服务端备份。
2. 多个浏览器标签同时编辑同一作品时采用最后一次保存生效；正式云端版需要版本号或乐观锁。
3. 图片像素处理和颜色匹配已迁移到可取消的 Web Worker，并保留不支持 Worker 时的同步回退；多页导出仍在主线程，大图在旧手机上导出时可能短暂卡顿。
4. MARD HEX 是屏幕近似值。色卡版本、测色来源和批次仍需由运营侧维护，不能由匹配算法替代实物校验。

## 云端社区启用条件

不要直接把当前本地点赞和发布草稿改成公开接口。正式服务端至少需要以下边界：

```text
Auth
  -> Project API (owner / visibility / revision)
  -> Community API (publish / like / save / remix)
  -> Moderation (report / review / takedown)
  -> Database (users / projects / pattern_versions / posts / reactions)
  -> Object Storage (source images / previews / exports)
```

最低数据约束：

- 所有写接口必须从服务端会话取得用户 ID，不能信任客户端提交的作者 ID。
- 项目更新携带 `revision`，使用乐观锁避免覆盖其他设备的新版本。
- 公开作品保存不可变的色卡版本 ID 和每格色号，不能只保存 HEX。
- 图片使用签名上传、MIME 与像素检查、大小限制和内容审核。
- 点赞和收藏按 `(user_id, post_id)` 建唯一索引，计数由数据库计算或可靠异步聚合。
- 删除采用软删除和审计时间；公开内容必须提供举报、屏蔽和下架流程。
- 数据库迁移先在预发布环境演练，并准备可验证的备份与恢复流程。

## 推荐上线顺序

1. 保持当前 App 本地优先版本，完成 TestFlight 真机验证。
2. 以 `.beadproject` v2 为基础设计跨设备云同步协议和服务端版本号。
3. 选择单一业务后端，不同时维护 Vercel 数据库和 Cloudflare D1 两套真源。
4. 先上线账号、私有项目同步和冲突处理，再开放公开发布。
5. 最后加入评论、关注、举报和店铺库存，避免在无治理能力时提前开放 UGC。
