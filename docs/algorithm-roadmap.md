# 算法路线图

## 当前算法

1. 读取用户上传图片。
2. 根据裁剪参数取得源区域。
3. 缩放到目标豆阵尺寸。
4. 逐格读取 RGB。
5. 将 RGB 转换为 CIE Lab。
6. 对店铺色卡逐色计算 Lab 欧氏距离。
7. 先做一次全色卡匹配并统计色号。
8. 取 Top N 色号作为色数上限。
9. 再把所有格子重映射到 Top N 色号。
10. 输出 `Pattern` 和用量表。

## 近期改进

### 1. 更稳定的色数限制

当前 Top N 按首次匹配用量决定。下一版可以加入：

- 颜色覆盖度评分
- 视觉重要度评分
- 肤色/黑白/高饱和色保护
- 用户锁定色号

### 2. Dithering 抖动

用于在色数少时保留渐变和阴影。

可选算法：

- Floyd-Steinberg
- Atkinson
- Bayer ordered dithering

界面上建议做成三档：

- 关闭
- 柔和
- 强化

### 3. 边缘保护

角色、文字、线稿图很依赖边缘清晰度。可以在缩放前做：

- 轻量锐化
- 边缘检测
- 暗线保护
- 轮廓色优先

### 4. 肤色保护

人物图最容易在肤色上翻车。建议加入：

- 肤色候选色集合
- 肤色区域检测
- 肤色区域限制匹配到指定色组
- 人工选择肤色基调

### 5. 背景处理

支持：

- 透明背景合成到指定底色
- 自动去白底
- 自动抠主体
- 背景不参与豆数统计

## 中期算法能力

### 区域编辑

- 框选
- 魔棒选择同色区域
- 批量换色
- 填充
- 锁色

### 替代色推荐

当某色缺货时：

1. 从同品牌同系列找最近色。
2. 从跨品牌映射表找等价色。
3. 按库存、成本和色差综合排序。

### 大图分板

用于大型挂画：

- 按模板板尺寸分区
- 分区坐标编号
- 每板色号用量
- 总用量汇总

## 评估指标

- 平均色差：所有格子的 Lab 差异均值。
- 最大色差：最不准的格子。
- 可制作度：色数、豆数、缺货色数量。
- 用户修改量：自动图纸生成后被手工改色的格子比例。
- 导出清晰度：打印时坐标和色号是否可读。
