# 拼豆社区产品设计

社区采用“作品优先”的轻量模型，不照搬 GitHub 的工程协作界面。

## 第一阶段

- 图片流：热门、最新、收藏与分类浏览。
- 作品详情：完整图纸、作者、尺寸、豆数、色卡和复刻说明。
- 喜欢与收藏：测试版先保存在当前设备。
- 一键复刻：复制到“我的作品”，自动转换为可编辑的 MARD 291 图纸。
- 来源标记：复刻作品持续显示原作者和原作名称。
- 更新记录：只展示作者的主要调整，不引入提交、分支等术语。
- 发布草稿：保存创作者昵称、作品说明与复刻权限，云端发布前仅存当前设备。

## 后续上线

- 登录、公开发布、关注作者与评论。
- 云端收藏、作品同步和真实的复刻数量。
- 原创声明、转载许可、举报和内容审核。
- 作品发布时保存色卡版本，避免同名色号跨品牌混用。

## 暂不加入

- Issue、Pull Request、分支合并等开发者工作流。
- 复杂的多人协同编辑。
- 没有明确授权的商业售卖与素材搬运。

产品重点是让用户看到好作品后，能够放心收藏、清楚识别来源，并快速进入自己的拼豆编辑流程。
