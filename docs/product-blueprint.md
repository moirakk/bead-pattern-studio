# 产品蓝图：拼豆创作平台

## 产品愿景

Bead Pattern Studio 不只是一个图片转拼豆图纸工具，而是一个围绕拼豆创作的完整平台：

```text
图纸生成工具 -> 个人作品集 -> 社群分享 -> 店铺与材料连接
```

目标是让用户从“看到一张喜欢的图片”一路走到“生成图纸、购买材料、完成作品、分享作品、再次创作”。

## 核心定位

为拼豆玩家、拼豆店主、手作创作者和图纸设计师提供一套完整的数字化创作工作流。

短期先做强工具；中期沉淀作品与用户；长期形成图纸、作品、材料和店铺的生态。

## 用户角色

### 拼豆新手

需求：

- 把图片快速变成能照着做的图纸。
- 不懂色号也能得到采购清单。
- 想参考别人做过的作品。

关键功能：

- 一键图片转图纸
- 新手友好的尺寸和豆数建议
- 打印/手机查看图纸
- 作品教程和模板推荐

### 进阶玩家

需求：

- 控制色数、尺寸、渐变、边缘和肤色。
- 手工精修图纸。
- 建立自己的图纸和作品库。

关键功能：

- 手工改单格、撤销/重做
- 区域编辑、锁色、抖动
- 私密/公开作品集
- 版本管理和二次编辑

### 拼豆店主

需求：

- 使用自己的色卡、库存和价格。
- 给客户传图定制报价。
- 生成材料包清单。

关键功能：

- 店铺色卡管理
- 库存与替代色
- 自动估价
- 客户传图入口
- 材料包导出

### 图纸设计师/创作者

需求：

- 发布图纸作品。
- 建立个人主页。
- 分享或售卖图纸。

关键功能：

- 图纸商城
- 作品展示页
- 免费/付费下载
- 水印和授权说明
- 社群互动

## 产品模块

### 1. 图纸工具

这是当前核心，也是平台的入口。

- 图片上传
- 裁剪与像素化
- 色卡匹配
- 色数限制
- 单格/区域编辑
- 撤销/重做
- PNG/PDF/CSV 导出
- A4 分页打印
- 材料清单

### 2. 图纸识别

将已有图纸或成品照片转成可编辑数字图纸。

- 识别像素图网格
- 识别纸质图纸坐标
- 识别成品照片中的豆阵
- 推断近似色号
- 输出可编辑 Pattern

优先级建议：先做“清晰像素图/网格图识别”，最后再做成品照片识别。照片识别受光照、角度、反光影响大，应作为高级能力。

### 3. 个人作品集

让用户有理由留下来。

- 我的图纸
- 我的成品
- 制作进度
- 私密草稿
- 公开作品
- 复刻/二创记录
- 色号和用量记录

### 4. 社群分享

让用户带来用户。

- 发布作品
- 点赞、收藏、评论
- 关注作者
- 标签和主题活动
- 图纸挑战
- 作品合集

社群不应一开始就做复杂。第一版只需要公开作品页、收藏和基础标签。

### 5. 店铺与材料

连接真实购买和商业闭环。

- 店铺色卡
- 库存
- 单价
- 缺货替代色
- 材料包清单
- 客户传图定制
- 报价单
- 订单导出

### 6. 图纸商城

当作品和图纸沉淀后再做。

- 免费图纸
- 付费图纸
- 作者主页
- 授权说明
- 平台抽成
- 店铺联名图纸

## 版本路线

### Phase 1：强工具

目标：让用户愿意用它生成图纸。

- 稳定图片转图纸
- 色卡匹配
- 手工编辑
- 撤销/重做
- A4 PDF
- CSV 用量表
- 公开网页

### Phase 2：保存与作品集

目标：让用户有资产沉淀。

- 用户账号
- 保存图纸项目
- 上传成品图
- 个人作品集
- 公开/私密切换
- 分享链接

### Phase 3：小程序

目标：国内传播和私域运营。

- 微信登录
- 手机端图纸查看
- 分享给好友/群
- 收藏图纸
- 上传作品
- 店铺二维码入口

### Phase 4：社群

目标：用户之间互相带动创作。

- 作品流
- 标签
- 评论
- 收藏
- 主题挑战
- 作者主页

### Phase 5：店铺版

目标：商业化和真实交易。

- 店铺后台
- 色卡/库存/价格
- 客户传图
- 自动报价
- 材料包清单
- 缺货替代

### Phase 6：图纸商城

目标：平台生态。

- 图纸发布
- 图纸售卖
- 作者分成
- 店铺材料包联动
- 授权和水印

## 商业模式

### 免费工具 + 高级导出

免费：

- 低分辨率图纸
- 基础色卡
- 基础 PNG/CSV

收费：

- 高清图纸
- A4 分页 PDF
- 商用模板
- 无水印
- 批量导出

### 店铺 SaaS

按月收费给拼豆店主：

- 店铺色卡
- 库存管理
- 报价工具
- 客户上传入口
- 材料包清单

### 图纸商城抽成

图纸作者售卖图纸，平台抽成。

### 定制服务

半自动定制图纸：

- 用户上传图片
- 工具生成初版
- 人工精修
- 交付专业图纸和材料清单

## 技术路线

### 当前

- Web MVP
- 浏览器 Canvas 处理图片
- 纯前端生成图纸和导出
- 核心算法已拆到 `lib/pattern`

### 下一步

- A4 PDF 导出模块
- IndexedDB 本地草稿
- 项目数据模型
- 用户账号
- 云端项目存储

### 中期

- API 服务
- 图纸项目数据库
- 对象存储保存图片和导出文件
- 小程序端复用核心算法
- 后台色卡/库存管理

## 产品原则

- 先做强工具，再做社区。
- 先让个人玩家觉得好用，再服务店铺。
- 核心算法保持平台无关，避免被 Web UI 绑死。
- 社区功能要围绕作品和图纸，不做泛社交。
- 商业化优先贴近真实制作流程：图纸、材料、报价、定制。

## 最近 90 天建议

1. 完成 A4 PDF 和更专业导出。
2. 支持项目保存和再次编辑。
3. 做移动端体验优化。
4. 做公开作品页原型。
5. 找 3-5 个拼豆玩家/店主试用，收集真实色卡和需求。
